# AI量化聊天区桌面重构设计（Final）

日期：2026-03-05

## 1. 目标

针对 AI 量化页面进行桌面端交互优化：
1. 聊天框初始更小，开聊后自动扩展
2. 支持多会话窗口与会话历史
3. 策略广场推荐策略至少 6 个
4. UI 风格保持与 Coinflux 现有设计一致

## 2. 范围约束

- 本期仅做桌面端，不做移动端专门适配。
- 保持现有业务链路（回测、部署门槛）不变，仅增强交互与布局。

## 3. 页面结构（桌面）

AI量化页改为双栏布局：
- 左栏：会话历史面板
  - 新建会话按钮
  - 会话列表（标题、最后更新时间）
- 右栏：聊天主区 + 策略广场 + 回测结果

## 4. 聊天框状态模型

### 4.1 高度状态
- `COMPACT`：初始小高度（约 240px）
- `EXPANDED`：开聊后大高度（约 520px）

### 4.2 触发规则
- 新会话且无消息 -> `COMPACT`
- 发送第一条消息后 -> `EXPANDED`
- 切回空白新会话 -> `COMPACT`

### 4.3 动画
- 高度变化采用 200~300ms 平滑过渡，避免视觉跳变。

## 5. 多会话设计

每个会话对象包含：
- `id`
- `title`
- `messages`
- `params`
- `backtestResult`
- `updatedAt`

交互：
1. 新建会话
2. 切换会话
3. 当前会话高亮
4. 标题默认来源：第一条用户消息截断（如前 16 字）

持久化：
- 本地存储键：`ai_quant_conversations_v1`

## 6. 策略广场（>=6）

最少 6 张推荐策略卡：
1. 价格动量（稳健）
2. 价格动量（进取）
3. 网格（区间震荡）
4. 布林带均值回归
5. 突破追涨（短周期）
6. 回撤抄底（中周期）

点击策略卡后：
- 应用参数到当前会话
- 在会话中写入“已应用策略模板”提示消息

## 7. 错误与空态

1. 无会话：显示新建引导
2. 本地数据解析失败：自动清理并回退空态
3. 回测不达标：保留“返回对话继续优化”路径

## 8. UI 约束

- 必须复用 Coinflux 现有视觉系统（颜色、边框、圆角、间距、文本等级）。
- 禁止引入新的主题体系。

## 9. 验收标准

1. 桌面端双栏布局生效。
2. 聊天框初始小、高度可自动扩展。
3. 支持多会话创建与切换。
4. 策略广场推荐策略不少于 6 个。
5. 回测不通过可继续对话优化。
6. 风格与 Coinflux 保持一致。
7. 本期不做移动端。
