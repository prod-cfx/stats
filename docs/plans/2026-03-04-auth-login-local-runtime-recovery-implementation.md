# 实施计划：Auth 登录本地运行恢复（无代码改动）

## 目标
恢复 `main` 下 OTP 登录本地可用，验证登录页样式与行为，不做任何业务代码修改。

## 步骤
1. 基线确认
- 切换到 `main` 工作区。
- 记录 `branch` 和 `sha`。

2. 启动 backend（3000）
- 使用仓库标准命令启动 backend dev。
- 校验 `3000` 监听可用。

3. 启动 front（3011）
- 使用 `main` 工作区启动 front dev 到 `3011`。
- 校验登录页可达：`/zh/auth/login`。

4. OTP 链路验证
- 验证发送验证码请求有成功响应。
- 验证验证码登录请求成功并建立会话。
- 若失败，记录接口返回体（路径/状态/错误消息）。

5. 样式验收
- 在白天模式检查登录卡片、文本、输入框、按钮、错误提示对比度。
- 截取结论：一致/不一致。

6. 证据回报
- 回报 `branch + sha + backend port + url + OTP 结果 + 样式结论`。

## 失败分支
- backend 启动失败：停止，回报错误并给出最小排查信息。
- OTP 失败：不改代码，先回报协议/配置层原因。

## 验收标准
- 能在 `main` 上稳定复现并完成 OTP 登录（发码+登录）。
- 结果可被同事用同样步骤重复。
