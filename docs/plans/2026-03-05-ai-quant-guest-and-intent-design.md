# AI量化未登录可浏览与登录续操作设计

## 1. 背景与目标
- 背景：当前 `/ai-quant` 未登录直接拦截，用户无法先浏览策略广场与对话入口。
- 目标：实现“未登录可浏览，操作需登录；登录后自动续操作”的前端闭环。
- 范围：MVP 前端优先，后端不新增匿名态协议。

## 2. 设计结论（已确认）
- 采用方案A：`return intent` 轻量续操作。
- 未登录用户可以看到：顶部轻量聊天框 + 精选策略广场（至少6个策略）。
- 未登录点击 `运行/编辑/发送`：统一提示登录，并记录意图后跳转登录。
- 登录成功返回 `/ai-quant` 后自动执行最近一次意图，执行后清理。

## 3. 信息架构
### 3.1 `/ai-quant` 未登录态（可浏览）
- 顶部：小高度聊天输入框（可输入草稿，发送受限）。
- 下方：精选策略卡片列表（运行、编辑双按钮）。
- 所有写操作均触发登录门禁。

### 3.2 `/ai-quant` 登录态（工作台）
- 左：会话列表（新建、切换、重命名、删除）。
- 右上：对话区（首屏小框，开始对话后自动扩展）。
- 右下：推荐策略（首次会话展示）。
- 回测结果与部署入口保持在工作台内，支持未达标回到对话继续优化。

### 3.3 `/ai-quant/plaza` 登录子页面
- 独立策略浏览、筛选、运行、编辑。
- 点击编辑可回流到对话页继续优化。

## 4. 关键交互流程
### 4.1 未登录点击“运行”
1. 保存意图：`{ type: 'run', strategyId, ts }`
2. 跳转登录：`/auth/login?redirect=/ai-quant`
3. 登录返回后自动运行该策略（先回测，再展示结果）
4. 清理意图

### 4.2 未登录点击“编辑”
1. 保存意图：`{ type: 'edit', strategyId, ts }`
2. 登录后自动导入策略参数到当前会话
3. AI 自动发送引导语，提示继续优化
4. 清理意图

### 4.3 未登录发送聊天
1. 保存意图：`{ type: 'chat', draft, ts }`
2. 登录后自动恢复草稿并触发发送
3. AI 继续追问缺失参数（交易所、币种、周期、仓位）
4. 清理意图

## 5. 意图模型与兜底
- 仅保留最近一条意图。
- TTL：30分钟，超时自动失效。
- 策略不存在：提示策略不可用，停留可浏览态。
- 自动续操作失败：降级手动模式，保留草稿并提示用户继续。

## 6. 权限与业务边界
- 未登录：只可浏览，不可执行写操作（发送、运行、编辑提交、部署）。
- 登录后：才允许会话写入、回测、部署。
- 部署前置：回测阈值（如最大回撤 <= 20%）保持不变。

## 7. 验收标准（MVP）
- 未登录访问 `/ai-quant` 可见轻量聊天框 + 至少6个精选策略。
- 未登录点击 `运行/编辑/发送` 必须触发登录流程。
- 登录后可自动续操作（run/edit/chat 三类链路至少各通过1条）。
- 登录工作台支持会话新建、切换、重命名、删除。
- 回测不通过后可在同一会话中持续优化至达标。
- UI 与 Coinflux 现有设计 token 保持一致。
