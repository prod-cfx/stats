# AI 量化前端优先版设计（Final）

日期：2026-03-05

## 1. 设计目标

本版本先做前端闭环，后端按协议逐步补齐。

目标主链路：
1. 用户登录后进入 AI 量化页面
2. 通过自然语言对话创建/优化策略
3. 查看后台回测结果
4. 回测不达标则继续对话优化
5. 回测达标后一键部署

## 2. 信息架构

### 2.1 导航
- 导航栏新增入口：`AI量化`
- 路由：`/[lng]/ai-quant`

### 2.2 页面结构（核心）
`/[lng]/ai-quant` 页面包含：
1. 顶部大 AI 对话框（核心入口）
2. 右上角快捷按钮：`配置交易所 API`
3. 对话框下方：策略广场（推荐策略卡）
4. 回测结果区（默认页内展开）
5. `全屏查看` 链接到独立回测页

### 2.3 回测详情页
- 路由：`/[lng]/ai-quant/backtest/[id]`
- 展示净值曲线、最大回撤、收益、交易次数与交易明细
- 保留一键部署主按钮（受门槛限制）

### 2.4 个人中心
在 `/{lng}/account` 增加 AI 量化分区：
- 我的策略（状态：DRAFT / BACKTESTED / RUNNING / ERROR）
- 回测记录
- 部署与运行日志
- 交易所 API 配置入口

## 3. 登录与权限规则

- 未登录不可使用 AI 量化功能。
- 未登录访问 `/{lng}/ai-quant` 显示拦截说明卡（不强制跳转）。
- 拦截卡提供 `立即登录` 按钮，登录后回到 AI 量化页。

## 4. 一键部署与 API Key 配置

### 4.1 部署前置检查
点击“一键部署”时按顺序校验：
1. 已登录
2. 已配置对应交易所 API Key（Binance/OKX）
3. 回测最大回撤达标

### 4.2 API Key 入口
- 主入口：个人中心 `交易所 API`
- 快捷入口：
  - AI 量化页右上角
  - 部署弹窗内 `去配置`

### 4.3 API Key 状态展示
- `未配置`
- `已配置`
- `校验失败`

前端仅展示掩码信息，完整密钥不回显。

## 5. 回测门槛与优化闭环

### 5.1 部署门槛
- 仅使用一条硬指标：`最大回撤 <= 20%`
- 若不满足：部署按钮禁用，不允许强制部署

### 5.2 不达标处理
- 提供主操作：`返回对话继续优化`
- AI 基于最新回测结果给调参建议
- 用户可持续“对话调参 -> 重新回测”，直到达标

## 6. 前端状态机

1. `UNAUTHENTICATED`：未登录拦截
2. `AUTHENTICATED_IDLE`：可开始对话
3. `DIALOGING`：参数补全/策略优化中
4. `BACKTEST_RUNNING`：回测执行中
5. `BACKTEST_FAILED_GATE`：回撤不达标，禁止部署
6. `BACKTEST_PASSED`：达标，可部署
7. `DEPLOYING`
8. `RUNNING`
9. `ERROR`

## 7. MVP 验收标准（前端）

满足以下即通过：
1. 导航有 `AI量化` 入口，页面结构为“大对话框 + 策略广场”。
2. 未登录时显示拦截卡，登录后可继续会话。
3. 回测结果支持页内展示与全屏查看两种方式。
4. 回撤 > 20% 时部署不可用，且可返回对话继续优化。
5. 达标后可触发一键部署流程（含 API Key 校验引导）。
6. 个人中心可查看 AI 量化相关策略、回测、部署和 API 配置状态。

## 8. 非目标（本阶段）

- 后端完整交易执行引擎重构
- 多策略自动组合与自动择优
- 复杂投顾评分系统

## 9. UI 设计约束（新增）

- AI 量化模块前端 UI 必须与现有 Coinflux 设计风格保持一致。
- 禁止引入新的主题体系或偏离现有视觉语言（颜色、字体、圆角、阴影、按钮样式、卡片间距）。
- 优先复用现有页面容器、组件和设计 token，确保用户在 AI 量化与现有模块之间切换时无视觉割裂。
