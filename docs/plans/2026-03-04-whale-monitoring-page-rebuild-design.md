# 巨鲸监控页重构设计（监控地址 + 实时巨鲸）

## 背景
当前监控页仍是通用规则卡片样式，和目标形态不一致。目标是对齐你提供的参考图：
- 监控地址：展示地址核心仓位指标 + 操作按钮。
- 实时巨鲸：展示与 Coinflux 实时巨鲸页面一致的表格内容，同时支持币种 + 金额阈值监控配置。

## 目标
1. 监控地址区域改为“指标表 + 操作栏”形态。
2. 实时巨鲸区域改为与现有实时巨鲸页面一致的视觉与列结构。
3. 保留现有登录校验与规则创建能力。
4. 使用现有交易统计弹窗能力，在监控页内直接打开。

## 非目标
- 不重做交易统计弹窗逻辑。
- 不改后端监控规则模型。
- 不做新的全局通知中心。

## 方案选择
采用：**专用 Section 组件拆分**（`AddressMonitorSection` + `RealtimeWhaleMonitorSection`）。

不采用：
- 单一 `MonitorSection` 继续硬扩展（可维护性差）。
- 整页复制实时巨鲸/持仓页面代码（重复过高）。

## 页面架构
- `NotificationsClient`：仅负责页面编排、弹窗状态、规则 refresh。
- `AddressMonitorSection`：
  - 数据：`ADDRESS` 规则 + 持仓指标数据。
  - 视图：地址与 5 项指标（总价值/未实现盈亏/可用保证金/保证金使用率/持仓）。
  - 操作：交易统计、关闭 Telegram、编辑、删除。
- `RealtimeWhaleMonitorSection`：
  - 表体：复用现有实时巨鲸表视觉与列结构。
  - 规则控件：币种 + 阈值金额创建 `SYMBOL` 规则。

## 关键交互
### 监控地址区
- 行操作按钮：
  - 交易统计：打开现有 `WhaleTradingStatsModal`（页内弹窗）。
  - 关闭 Telegram：写回 `channels.telegram=false`。
  - 编辑：打开创建弹窗的编辑态。
  - 删除：移除监控规则。

### 实时巨鲸区
- 展示列：地址、币种、持仓价值、开盘价、时间。
- 配置规则：用户选择币种并输入阈值金额后创建监控。

## 数据流
1. 规则数据来自 `useWhaleNotificationRules`。
2. 地址指标来自鲸鱼持仓页现有数据接口（按地址拉取）。
3. 交易统计复用现有弹窗组件与数据请求。
4. 规则创建/更新/删除复用 `whale-notification-api`。

## 异常与降级
- 规则接口 `404/405`：走本地 fallback（已确认策略）。
- 指标接口单地址失败：该行显示 `-`，不影响其他行。
- 交易统计失败：弹窗内显示空态，不关闭。
- `401/403`：维持严格鉴权失败，不伪成功。

## 验收标准
1. 监控地址区包含列与操作：
   - 地址、永续合约总价值、未实现盈亏、可用保证金、保证金使用率、持仓。
   - 按钮：交易统计 / 关闭 Telegram / 编辑 / 删除。
2. 交易统计在监控页内弹出。
3. 实时巨鲸区表体视觉与现有实时巨鲸页一致。
4. 实时巨鲸区可配置币种 + 阈值金额监控规则。
5. 页面在浅色/深色主题下均保持 Coinflux 风格一致。
