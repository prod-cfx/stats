# 账号设置与 AI量化分离设计（Frontend）

日期：2026-03-05

## 1. 目标

将“账号设置”和“AI量化”在用户中心内彻底分离，避免混排造成认知负担。

核心要求：
1. 右上角用户区改为下拉菜单，包含 `账号设置`、`AI量化`、`登出`
2. 进入账号页后，以 Tab 分区展示：`账号设置` 与 `AI量化`
3. 两个 Tab 内容独立，不再混在同一布局块中

## 2. 路由与状态

- 入口路由保持：`/{lng}/account`
- Query 参数：`tab`
  - `tab=settings`
  - `tab=ai-quant`
- 默认行为：无 `tab` 或非法值时回退到 `settings`

## 3. 导航与交互

### 3.1 顶部右上角下拉
登录后点击用户身份区域，弹出菜单：
- `账号设置` -> `/{lng}/account?tab=settings`
- `AI量化` -> `/{lng}/account?tab=ai-quant`
- `登出`

### 3.2 账号页 Tab
账号页顶部新增 Tab 条：
- `账号设置`
- `AI量化`

点击 Tab 时仅切换当前 Query 与展示面板，不做整站跳转。

## 4. 页面内容归属

### 4.1 settings 面板（仅账号）
- Telegram/邮箱绑定
- 账号信息（userId 等）
- 登出按钮

### 4.2 ai-quant 面板（仅 AI量化）
- 进入 AI量化主页面入口
- 交易所 API 配置区
- AI量化策略/运行入口卡片（MVP 先保留入口，后续接真实数据）

## 5. 错误处理与兼容

1. 未登录访问账号页：保持现有登录重定向逻辑
2. 非法 tab：自动回退 `settings`
3. 旧入口兼容：`/{lng}/account` 仍可用，默认显示账号设置

## 6. UI 约束

- 严格复用 Coinflux 现有设计变量与组件风格
- 不引入新的主题体系
- 保持桌面与移动端导航体验一致

## 7. 验收标准

1. 右上角下拉可见三个菜单项：账号设置、AI量化、登出
2. 点击 `账号设置` 进入 `tab=settings`，仅显示账号内容
3. 点击 `AI量化` 进入 `tab=ai-quant`，仅显示 AI量化内容
4. `/{lng}/account` 无 tab 默认进入 settings
5. 非法 tab 参数自动回退 settings
6. 整体视觉风格与 Coinflux 当前 UI 一致
