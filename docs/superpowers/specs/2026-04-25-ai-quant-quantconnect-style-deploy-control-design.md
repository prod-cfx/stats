# AI Quant 类 QuantConnect 部署控制方案

## 摘要

AI Quant 策略部署采用类 QuantConnect 的运行控制模型：
策略运行中不能直接修改；用户必须先停止策略，必要时处理持仓和挂单，
然后再修改并重新部署当前已发布版本。

本方案保持当前“单运行实例”的部署模型。它不引入“一份策略部署多个独立运行实例”，
也不把“恢复旧版本”做成主要产品路径。

## 目标

- 让 AI Quant 页面体验与当前后端模型对齐：部署会复用一个绑定到发布快照的策略实例。
- 避免用户误以为重复点击部署会创建多个独立运行策略。
- 阻止用户在策略运行中直接修改核心策略。
- 提供完整停止流程，包括持仓和挂单处理。
- 在当前测试环境中支持 OKX 模拟盘真实撤单和平仓。
- 停止后统一使用“重新部署”语义：
  - 如果策略没有修改，重新部署就是运行同一个已发布版本。
  - 如果策略已修改并重新发布，重新部署就是运行新发布版本。

## 非目标

- 不支持一份策略部署多个独立运行实例。
- 本阶段不做版本历史选择器。
- 不把“恢复旧版本”作为主路径。
- 本阶段不做实盘账户级别的完整强风控；当前设计基于 OKX 模拟盘测试环境。

## 当前项目上下文

当前实现已经具备本方案需要复用的核心基础：

- `StrategyInstance.status` 支持 `draft`、`running`、`stopped`。
- 部署请求通过 `DeployRequest` 做用户维度的 `deployRequestId` 幂等。
- AI Quant 部署绑定 `publishedSnapshotId`。
- 部署 repository 路径复用发布快照来源的策略实例，而不是创建新的策略实例。
- 策略详情 DTO 已经包含持仓相关信息。

方案必须继续保护 AI Quant 主数据流：

`自然语言 -> clarification gate -> canonical spec v2 -> semantic view(specDesc)
-> confirm canonical snapshot(canonicalDigest) -> IR -> AST -> compiled script
-> publish -> publishedSnapshotId -> backtest(using publishedSnapshotId)
-> report -> deploy(using publishedSnapshotId)`。

## 产品模型

### 用户可见状态

- `未部署`：已有可部署发布快照，但没有运行实例。
- `运行中`：策略已经部署并正在执行；核心编辑被阻止。
- `停止中`：停止或“平仓并停止”正在处理中；禁止重复危险操作。
- `已停止`：运行实例已停止；用户可以编辑或重新部署。
- `草稿已修改`：用户停止后修改了策略，需要重新确认、发布、回测后再重新部署。
- `可重新部署`：当前发布快照满足重新部署条件。

### 按钮语义

- 从未部署且回测通过：`一键部署`
- 正在运行：`已部署运行`，按钮禁用，同时显示 `查看运行策略`
- 运行中用户尝试编辑：弹框拦截
- 已停止，当前快照可部署：`重新部署`
- 已停止，草稿已修改但未发布：`继续编辑` 或 `生成并发布`
- 已停止，新快照已发布且回测通过：`重新部署`

产品上不把“恢复旧版本”作为一等路径。  
如果用户停止后没有修改，`重新部署` 实际上就是恢复同一版本运行。  
如果用户停止后修改并发布，`重新部署` 就是运行最新发布版本。

## AI Quant 对话页流程

### 运行中编辑拦截

当策略正在运行，用户点击 `返回对话修改` 时，不进入编辑态，而是弹框。

弹框标题：

> 策略正在运行，不能直接修改

弹框正文：

> 当前策略正在使用已部署版本运行。为避免运行逻辑与编辑内容不一致，
> 请先停止策略。停止后你可以修改策略并重新部署。

按钮：

- `查看运行策略`
- `停止策略`
- `取消`

`查看运行策略` 跳转到账户策略详情页。  
`停止策略` 打开停止流程。

### 回测结果卡片的已部署状态

如果当前对话关联的策略已经部署并处于运行中：

- 原部署 CTA 替换为禁用态 `已部署运行`。
- 显示 `查看运行策略`。
- 不允许从该状态重复部署。
- 如果发布快照需要重新发布或缺少运行绑定真相，继续沿用现有 fail-closed 行为。

## 停止策略流程

用户点击 `停止策略` 后，系统必须先拉取最新策略详情，再展示确认弹框。
确认弹框需要基于最新状态、持仓概览、挂单概览渲染。

### 无持仓、无挂单

弹框标题：

> 确认停止策略？

弹框正文：

> 停止后策略不再执行，也不会产生新的交易信号。

按钮：

- `确认停止`
- `取消`

成功后策略状态变为 `已停止`，提示：

> 策略已停止。你现在可以编辑策略，或稍后重新部署。

### 存在持仓或挂单

弹框标题：

> 当前策略仍有持仓或挂单

展示当前风险摘要，字段尽量包括：

- 交易所账户
- 交易对
- 市场类型
- 当前方向
- 当前仓位数量
- 当前挂单数量
- 当前浮盈亏，如果接口可提供

按钮：

- `仅停止，保留持仓/挂单`
- `平仓并停止`
- `取消`

#### 仅停止，保留持仓/挂单

行为：

- 停止策略。
- 不撤销挂单。
- 不关闭持仓。
- 成功后提示：

> 策略已停止。现有持仓和挂单仍然保留，需要你单独管理。

#### 平仓并停止

当前测试环境行为：

- 使用 OKX 模拟盘集成撤销该策略相关未成交挂单。
- 使用 OKX 模拟盘集成关闭该策略相关持仓。
- 撤单和平仓成功后，再停止策略。
- 返回最新策略详情。

失败处理：

- 如果撤单或平仓失败，不要宣称策略已完整停止，除非后端能证明当前状态安全。
- 展示失败原因，并让用户留在运行策略控制界面。
- 处理中必须禁用重复点击，或后端保证幂等，避免重复平仓。

## 重新部署流程

当策略已停止，并且当前发布快照可部署时，显示 `重新部署`。

确认弹框标题：

> 重新部署策略？

弹框正文：

> 系统将按当前已发布版本重新部署策略，并开始运行。

如果当前发布快照与上次运行快照不同，追加提示：

> 本次将运行最新发布版本，旧运行版本不会继续执行。

按钮：

- `确认重新部署`
- `取消`

重新部署继续使用现有 deploy 接口和 `publishedSnapshotId` 绑定。  
后端继续保持当前单策略实例语义。

## 策略详情页

策略详情页作为运行控制中心。

### 运行中

展示：

- 状态：`运行中`
- 主操作：`停止策略`
- 风险操作：`平仓并停止`
- 次操作：`返回对话修改`，但点击后仍然打开运行中编辑拦截弹框

### 已停止

展示：

- 状态：`已停止`
- 主操作：`重新部署`
- 次操作：`返回对话修改`

### 处理中

展示：

- 状态：`处理中`
- 禁用重复停止、平仓、重新部署操作

## 后端契约

### 策略详情

策略详情接口需要返回足够状态，供前端安全决策：

- 策略状态。
- 当前 `publishedSnapshotId`。
- 运行绑定状态和错误码。
- 持仓概览。
- 挂单概览。
- 是否允许停止。
- 是否允许平仓并停止。
- 是否允许重新部署。

如果当前还没有挂单概览，至少补充最小字段：

- 挂单数量
- 交易对级别摘要

在这些字段可用前，不应开放 `平仓并停止`。

### 停止策略

现有 action 接口可以继续支持 `stop`。

要求：

- 停止策略执行。
- 不平仓。
- 不撤单。
- 返回最新策略详情。

### 平仓并停止

新增明确的账户策略动作：`liquidate_and_stop`。

该动作仍走现有策略 action 控制面，但必须和普通 `stop` 区分开。
平仓逻辑不能隐藏在普通 `stop` 后面。

要求：

1. 解析当前用户和策略。
2. 确认策略属于当前用户。
3. 确认策略正在运行，或处于允许平仓并停止的状态。
4. 解析该策略相关未成交挂单和持仓。
5. 通过 OKX 模拟盘撤销该策略相关挂单。
6. 通过 OKX 模拟盘关闭该策略相关持仓。
7. 停止策略。
8. 返回最新策略详情。

幂等要求：

- 处理中重复请求不能重复发出平仓单。
- 如果相关持仓和挂单已经为空，可以直接进入停止策略步骤。

### 重新部署

继续使用当前 deploy API，并继续传入 `deployRequestId` 和
`publishedSnapshotId`。

要求：

- 复用现有单策略实例。
- 将运行绑定到当前 `publishedSnapshotId` 和 snapshot hash。
- 按当前实现初始化 runtime execution states。
- 部署成功后将策略状态置为 `running`。

## 错误处理

- 停止失败：展示失败原因，策略保持 `运行中`。
- 平仓失败：展示失败原因，不宣称策略已停止。
- 部分平仓：展示部分成功结果，允许重试。
- 重新部署失败：保持 `已停止` 状态，展示失败原因。
- 快照缺失或需要重新发布：提示重新生成并发布策略。
- OKX 模拟盘凭证缺失：引导用户去交易所 API 配置。

## 测试要求

### 前端测试

- 运行中策略点击返回编辑会被拦截，并显示弹框。
- 运行中策略显示 `已部署运行`，不显示可点击部署 CTA。
- 无持仓/挂单时，停止流程展示简单确认。
- 有持仓/挂单时，停止流程展示“仅停止”和“平仓并停止”选项。
- 已停止策略允许进入编辑。
- 已停止且可部署策略显示 `重新部署`。
- 当前发布快照变化时，重新部署确认弹框展示新版本提示。

### 后端测试

- `stop` 不平仓、不撤单。
- `liquidate_and_stop` 先撤销策略相关挂单，再关闭策略相关持仓。
- `liquidate_and_stop` 只有在撤单和平仓成功后才停止策略。
- `liquidate_and_stop` 对重复请求幂等。
- 重新部署保持单实例语义。
- 重新部署继续绑定 `publishedSnapshotId` 和 snapshot hash。

### 集成测试

- OKX 模拟盘路径可以关闭测试仓位，然后停止策略。
- OKX 模拟盘失败会被透出，不会错误地把策略标记为已停止。

## 上线说明

本方案适用于当前 OKX 模拟盘测试阶段。  
未来如果要开放到真实实盘账户，需要额外增加：

- 更强的平仓二次确认。
- 滑点和手续费提示。
- 持仓、挂单归因保证。
- 停止和平仓操作审计日志。
- 停止和平仓的独立权限控制。
